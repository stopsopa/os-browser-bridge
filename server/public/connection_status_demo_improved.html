browser subscribes to event 'os_browser_bridge_connection_status'

This is improved version of connection_status_demo.html. Here we can just subscribe to he event 'os_browser_bridge_connection_status' which from now on is guaranteed to be emitted immediately after page is loaded

WARNING: If you planning to listen to os_browser_bridge_connection_status event then subscribe to it before wating for waitForConnectionStatus() because initial event os_browser_bridge_connection_status will arrive before waitForConnectionStatus() 

        /* subscribe before waitForConnectionStatus() */
        document.addEventListener("os_browser_bridge_connection_status", (event) => {
          const {
            type, // "os_browser_bridge_connection_status"
            detail: {
              isConnected, // boolean
              details: {
                state, // "connected"
              },
            },
          } = event;
        });
